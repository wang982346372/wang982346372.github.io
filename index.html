Linux介绍

Linux操作系统

1. Linux介绍

硬件：五大部件（运算器、控制器、存储器、输入设备、输出设备） —> 裸机
软件：系统软件和应用软件

操作系统：覆盖在硬件上的软件，用于提供人和计算机交互的接口（界面）

个人操作系统：Windows / macOS / Linux
服务器操作系统：商业化的Unix / Solaris / Linux / Windows Server

Unix ---> Minix ---> Linux ---> Android
Unix ---> BSD ---> FreeBSD ---> macOS ---> iOS
Unix ---> Solaris / Hp-Unix / AIX

Linux ---> Redhat ---> CentOS
远程连接 ---> SSH ---> XShell / FinalShell

没有GUI（图形用户界面） ---> Shell（壳程序 - 人机接口）

kernel ----> Linux发行版本
Redhat / CentOS / Ubuntu / Debian / SUSE

2. 指令

命令格式：命令 [参数] [作用的对象]

调出帮助文档
\1. 在命令后面加上–help参数
\2. 使用man命令获取帮助手册 —> manual
man mkdir

查看历史命令：history
清除历史命令：history -c
执行历史命令：!历史命令编号

文件操作相关的命令
pwd —> print working directory —> 显示当前工作目录
cd —> change directory —> 切换目录（绝对路径/相对路径）
ls —> list directory contents —> 列出文件夹下的内容
-l：长格式查看
-a：查看所有文件（包括以点开头的隐藏文件和文件夹）
mkdir —> make directories —> 创建文件夹
-p：创建父文件夹（如果不存在）
rmdir —> remove directory —> 删除空文件夹
touch —> 创建空文件或者修改文件的最后访问时间
rm —> remove —> 删除文件
-f：（force）强制删除
-r：（recursive）递归式删除（可以用来删除文件夹）
cat —> concatenate —> 查看文件的内容
-n：显示行号
head / tail —> 查看文件的开头和结尾
-行数
more / less —> 逐页逐行查看文件内容
wc —> word count —> 统计行数、单词数、字节数
iconv —> 转换文件编码

mv —> move —> 移动文件 —> 剪切
cp —> copy —> 拷贝文件
-r：（recursive）递归式拷贝（拷贝文件夹）

file —> 查看文件属性

压缩文件：
gzip / gunzip
xz -z / xz -d

WinRAR —> 归档/解归档
WinZip —> 压缩/解压缩

归档：把多个文件合并到一个文件中
解归档：把一个文件拆成多个文件
tar -cf html.tar *.html
tar -xf html.tar

scp root@一台服务器的IP地址:/路径/文件名 \ root@另一台服务器的IP地址:/路径/文件名

unicode—> 字符集（万国码）—> 其中一种实现方式叫utf-8

常用快捷键：
Tab：自动补全命令或路径
Ctrl+u：删除整行
Ctrl+w：删除光标前面的单词
Ctrl+a：光标回到行首
Ctrl+e：光标去到行尾

重定向：
输出重定向：>
echo apple > fruits.txt
追加输入重定向：>>
echo apple >> fruits.txt
echo banana >> fruits.txt
错误输出重定向：2>
输入重定向：<

管道：| 操作系统提供的进程间通信的I/O通道
前一个命令（进程）的输出作为下一个命令（进程）的输入

sort fruits.txt | uniq -c

3. Web服务器

Web服务器：Apache / Nginx
启动Nginx服务（Web服务） —> Web服务器
systemctl start nginx (CentOS 7.x)
service nginx start (CentOS 6.x)
停止Nginx服务
systemctl stop nginx
重启
systemctl restart nginx
查看运行状态
systemctl status nginx
设置/禁用开机自启
systemctl enable nginx
systemctl disable nginx

查看网络状态 - netstat -ntlp
-n：地址和端口的数值
-t：tcp
-l：listen - 正在监听
-p：process - 显示进程信息

CentOS系统上如何安装软件和服务
\1. 使用包管理工具安装软件
yum –> Yellow dog Updater Modified
搜索：yum search nginx
安装：yum install -y nginx
卸载：yum erase -y nginx / yum remove -y nginx
更新：yum update nginx / yum update
查看信息：yum info nginx
查看安装的所有软件包：yum list installed

6. 权限

-rw-r--r--

第一个 - 表示文件 d 表示文件夹 l 链接 表示快捷方式

r w x 表示 r 读权限 w 写权限 x 执行权限 - 表示没有对应的权限

第一组 表示文件所有者

第二组 表示文件所有者同组用户

第三组 表示其他用户

9. git的使用

在所需要版本控制的文件夹中执行 git init建立仓库

执行代码 git status 查看当前状态

执行代码 git add 文件 将需要版本控制的文件加入缓存区

需要把所有的文件都加入到缓存区 git add .

确定需要加入到本地仓库中执行 git commit -m "项目提交的理由" 提交代码

第一次提交需要设置 邮箱 用户

git config --global user.email '982346372@qq.com'

git config --global user.name 'wangchengmin'

输入 git log 查看提交记录 git reflog

git restore 文件名 恢复 文件

git reset --hard 版本号 退回到版本号当时的内容 版本号在 git log中查看

添加远端服务器git remote add origin git@gitee.com:wang_chengmin/hellogit.git

输入 git remote -v 查看远端仓库

git remote remove origin 删除远端服务器

同步代码

上推(把自己的代码同步到远端服务器)

git push -u origin master

下拉

git pull

查看冲突

git diff

查看后重新提交

配置免密登录

ssh-keygen -t rsa -b 2048 -C "982346372@qq.com"

-t 是指定算法 -b 指定长度

会在/目录生成隐藏文件 .ssh文件夹

.ssh 中 id_rsa 是本地密匙

将 id_rsa.pub中的内容复制到 远端服务器中

删除原有远端服务器

连接远端服务器 ssh连接

git remote add origin git@gitee.com:wang_chengmin/hellogit.git

git push -u origin master

重新建立免密连接

删除文件后从远端恢复

git clone --depth=1 git@gitee.com:wang_chengmin/hellogit.git code

–depth=1 是只下载最新版本 code 是指定文件夹

查看分支

git branch

git branch -a

创建分支

git branch 分支名

切换分支

git switch 分支名

合并分支

git merge 分支名

删除分支

git brench -d 分支名

git brench -D 分支名

提交新分支 建立关联

git push -u origin hello: develop

hello是新的分支 ：后是推上远端服务器的新名字
